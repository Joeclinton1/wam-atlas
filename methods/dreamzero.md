  437: 3.2. Real-time Execution of DreamZero
  438: Diffusion-based WAMs inherit powerful generalization from video foundation models, but their iterative
  439: denoising process creates a fundamental tension with reactive robotic control. We address two questions: (1)
  440: What prevents WAMs from being reactive policies? (2) How do we resolve this for real-time control?
  441: 
  442: 3.2.1. The Reactivity Gap
  443: Reactive policies must respond to environmental changes within tens of milliseconds. A naive implementation
  444: of DreamZero on a single GPU requires approximately 5.7 seconds per action chunk due to three bottlenecks:
  445: (1) iterative denoising across 16 diffusion steps required for smooth actions, (2) the computational cost of a
  446: 14B parameter DiT backbone, and (3) sequential execution that blocks robot motion during inference. This
  447: latency makes closed-loop control infeasible.3

  449: 3.2.2. Asynchronous Closed-Loop Execution
  450: Our first step towards resolving this is through asynchronous execution that decouples inference from action
  451: execution. Rather than waiting for each inference to complete, the motion controller continuously executes the
  452: most recent action chunk while inference runs concurrently on the latest observation. This structure transforms
  453: the latency constraint from “inference must complete before the robot moves” to “inference must complete
  454: before the current action chunk expires.” In our experiments, we deploy policies at an action horizon of 48
  455: steps at 30Hz control frequency (1.6 seconds per chunk) for bimanual manipulation robots. Hence, we target
  456: inference latency below approximately 200ms to ensure sufficient overlap for smooth, reactive control.

 1098: Q1. Does data diversity improve generalization?
 1099: We compare DreamZero trained on 500 hours of diverse data versus 500 hours of repetitive data, where
 1100: the latter contains 70 tasks with many repeated demonstrations per task using similar object positions and
 1101: configurations. As shown in Table 4, diverse data substantially improves generalization (33% → 50%), even on
 1102: simple pick-and-place tasks. We hypothesize this reflects WAMs’ learning dynamics: since video prediction
 1103: is largely inherited from pretraining, the key challenge is learning inverse dynamics. A robust IDM requires
 1104: diverse state-action correspondences across varied contexts, which repetitive data inherently lacks.
 1105: 
 1106: Q2. Does WAM performance scale with model size?
 1107: For VLAs, scaling model size improves semantic reasoning but not necessarily action prediction. We find that
 1108: WAMs exhibit clearer scaling behavior: the 14B model significantly outperforms the 5B model (50% vs. 21%),
 1109: with the smaller model prone to visual hallucinations that propagate to erroneous actions.

 1126: diverse data boosts downstream performance in Table 4. However, we still have lacking evidence for scaling
 1127: laws for robot foundation models, specifically for WAMs. Similar to scaling laws for language models (Kaplan

 1129: to be explored to determine the optimal configuration to extract the maximal capability of WAMs. We expect
 1130: that the tendency of scaling WAMs to be different from VLAs, showing a more direct scaling law for actions.
