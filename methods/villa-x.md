  818: In this appendix, we provide extended information on the architecture, training protocols, and inference
  819: behavior for our Latent Action Model (LAM).
  820: 
  821: A.1     Architecture Overview
  822: Our LAM comprises four main modules:
  823:     (i) Spatial–Temporal Transformer (ST-Transformer) Inverse Dynamics Model (IDM): Takes a video
  824:         clip (by default, 8 × 224 × 224) as input. We employ patch embedding with a patch size of 14 and
  825:         stack 12 ST-blocks [65], each with a hidden dimension of 768 and 32 attention heads.
  826:  (ii) Vector Quantization (VQ) Module: Maps the continuous IDM outputs to discrete latent tokens,
  827:       each associated with a codebook entry. We set the codebook size to 32. While the model inter-
  828:       nally uses discrete token indices during training, the continuous codebook centers are used in
  829:       downstream modules.
  830:  (iii) Image Reconstruction Forward Dynamics Model (FDM): A 12-layer Vision Transformer (ViT)-base
  831:        network that takes the current frame ot and a latent action zt to predict ôt+K .
  832:  (iv) Proprioceptive Forward Dynamics Model (proprio-FDM): A 2-layer MLP with dual output heads
  833:       to predict future robot states q̂t+i and low-level robot actions ât+i . This module takes the current
  834:       robot state qt , the latent zt , and an embodiment context vector ce .
  835: Rather than predicting a single latent action per pair of frames, the ST-Transformer-based IDM processes
  836: a sequence of TLAM frames, resulting in TLAM − 1 latent tokens. We use TLAM = 8. By reconstructing
  837: future frames with the FDM and future states/actions with the proprio-FDM, the model learns a latent
  838: representation that is both visually and physically grounded.
  839: 
  840: A.2     Training Details
  841: We train our LAM on a combination of human egocentric data (e.g., Ego4D [21]) and robot trajectories
  842: (e.g., OpenX [12]). Samples lacking low-level robot annotations (e.g., human videos) exclude the proprio-
  843: FDM branch, using only the visual FDM objective.
  844: 
  845: Hyperparameters. We use a batch size of 512 and a learning rate of 1.5 × 10−4 , with a 2000-step linear
  846: warmup. Training lasts approximately 4 days on 128 NVIDIA A100 GPUs. Both the visual FDM and
  847: proprio-FDM share the same weighting in the overall loss. Throughout training, each latent token is
  848: discretized via the VQ module but is represented by its continuous codebook center in subsequent
  849: network components.

  854: examine whether the learned latent tokens accurately capture future frame content, robot states, and
  855: actions. This reconstruction-based analysis aids in understanding and debugging the physical grounding
