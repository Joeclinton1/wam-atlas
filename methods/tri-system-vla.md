  785: Ablation Study
  786: To analyze our system’s ability to achieve out-of-distribution generalization (e.g.,
  787: the unseen left-cup task), we conduct a structured analysis of Table 2.
  788: Q1: What enables OOD execution? The 70% success rate in Case 2
  789: (vs. 0% in Case 1) demonstrates that subtask-level training allows System One
  790: to learn shared representations. By including left-arm data for other objects
  791: (bowls), System One successfully transfers manipulation skills to the OOD cup.
  792: Q2: Why do dual-system baselines consistently fail on this task?
  793: Baselines lack temporal memory and heuristic logic, suffering from Execution
  794: Stagnation. After placing a bowl on the right, the right arm remains centered
  795: and visually proximal to the left cup. The policy repeatedly attempts unfeasible
  796: reaches with the right arm, becoming trapped in a kinematic loop.
  797: Q3: How do human-inspired rules resolve this? Our Tri-System uses
  798: System Three to detect stagnation and trigger a robot state reset. Retracting
  799: the arm breaks the visual-kinematic trap, revealing that the left cup is closer to
  800: the left arm and enabling a successful hand-over or replan.
  801: Q4: How do prompt formulations impact performance? Case 4’s
  802: 90% success rate proves that structured prompts (e.g., “...with <arm>”) sig-
  803: nificantly enhance System One’s OOD execution by providing precise spatial-
  804: morphological grounding. However, in Case 3 (30%), using an identical prompt
  805: generated by System Two actually degrades performance. This suggests that
  806: while System One possesses the latent capacity for OOD tasks, the current
  807: System Two (VLM) suffers from distributional overfitting, failing to correctly
  808: predict the <arm> and <side> tokens for unseen scenarios. This identifies Sys-
  809: tem Two’s reasoning as the primary bottleneck, suggesting that more powerful
  810: VLMs could further bridge this gap.
